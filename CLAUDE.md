## 项目概述

顺丰快递月度费用核对 Web 应用。接受 8 个 Excel 文件（账单/订单/城市/价格/B站赠送/KOC/出入库/售后），自动匹配店铺、省份，计算运费，与账单应付金额核对，输出结果。

## 启动命令

```bash
cd sf-freight-check
python app.py
# 访问 http://127.0.0.1:5000
```

## 依赖

- Python 3.x
- Flask
- openpyxl

## 核心架构

### 8个上传文件的角色

| 文件 | 作用 |
|------|------|
| 顺丰月结账单 | 主数据源，含运单号码/到件地区/产品类型/计费重量/应付金额 |
| 订单明细表 | 快递单号→店铺名称 |
| 全国城市省份表 | 城市→省份映射 |
| 顺丰物流价格表 | 按省份+重量段计算运费 |
| B站赠送明细表 | 发出快递单号→型号（B站抽奖渠道） |
| 推广KOC明细表 | 发出/寄回快递单号→型号（KOC渠道） |
| 出入库单明细表 | 出入库单号→备注（[其他]吉他情报局渠道） |
| 售后单明细表 | 退回快递单号→店铺名称 |

### 核心处理流程（cleaning.py）

1. **到件地区清洗** `normalize_area()` — 按 `/、,，-` 拆分多城市地区，取主城市
2. **寄件地区清洗** — 同上，取主城市用于省份推断
3. **级联店铺匹配** `cascade_shop_match()` — 依次尝试：订单明细→售后→KOC→B站
4. **省份推断** `guess_province()` — 优先查城市→省份映射表，兜底用 `PROVINCE_HINTS`（全称）；再用 `normalize_province()` 统一短名/全称
5. **运费计算** `calc_freight()` — 核心，见下方分产品规则
6. **账单核对** `process_bill()` — 对每行追加 店铺/省份/运费/总运费/是否一致/是否异常/备注

### 运费计算规则（按产品类型）

**顺丰干配**：`calc_freight(..., '顺丰干配', ...)`
- 按"到达省份"匹配 `顺丰干配` Sheet
- 公式：`首重价格 + (计费重量 - 首重重量) × 续重倍率`
- 特殊：西藏 → 使用标快异地价格

**顺丰标快/特快**：`calc_freight(..., '顺丰标快'/'顺丰特快', ...)`
- 先推断寄件省份（寄件地区主城市 → 省份）
- 再推断到件省份（到件地区主城市 → 省份）
- 判断同城（同省）vs 异地：
  - 同城 → 查 `顺丰标快（同省）` Sheet（寄件省份=广东）
  - 异地 → 查 `顺丰标快（大陆地区异地）` Sheet
- 匹配方式：寄件省份 + 到件城市（在城市列表中模糊匹配，支持 `/、,` 分隔）
- 公式：`首重价格 + (计费重量 - 首重重量) × 续重单价`
- 重量区间：按 `续重重量1（＜N）` 和 `续重重量2（≥N）` 的阈值决定用哪个续重单价；若 `续重单价2 = /`，则沿用续重单价1
- 兜底：寄件省份在异地 Sheet 中无匹配时，尝试用**目的地省份**再查

**暂不处理**：服务=转寄/退回、保价、包装服务、顺丰特快 → 运费留空

### 省份名规范化

`PROVINCE_SHORT_TO_FULL` / `PROVINCE_FULL_TO_SHORT` 处理短名↔全称互换，确保匹配到价格表。价格表中寄件省份字段用全称（如"广东省"），城市省份表用短名（如"广东"）。

### 输出新列

`店铺 / 省份 / 运费 / 上浮费 / 总运费 / 是否一致？ / 是否异常？ / 备注`
